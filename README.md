# Pizza Store

Современное Single Page Application для заказа пиццы. Проект демонстрирует работу с удаленным API, сложную фильтрацию и сортировку, управление глобальным состоянием через Redux Toolkit, синхронизацию интерфейса с адресной строкой браузера, а также сохранение корзины в localStorage.

## Стек

- Core: React 19 (Hooks)
- TypeScript: Полная типизация проекта для повышения надежности и поддерживаемости кода
- State Management: Redux Toolkit (Slices, AsyncThunk для асинхронных запросов) с типизированными экшенами и селекторами
- Routing: React Router v7 (Синхронизация фильтров с URL через qs)
- Styling: SCSS, CSS Modules, Адаптивная верстка
- API Layer: Axios (Интеграция с MockAPI)
- Оптимизация:
  - react-content-loader - Skeleton UI для комфортного ожидания загрузки.
  - react-paginate - удобная навигация по страницам каталога.
  - lodash.debounce - минимизация лишних запросов при поиске.
- Инструменты: Vite, nanoid, Tabler Icons.

## Ключевые архитектурные решения:

### 1. Строгая типизация

TypeScript используется во всем проекте - от моделей данных до компонентов и глобального состояния:

- Строгая типизация Redux store с использованием `RootState` и `AppDispatch`
- Типизированные хуки `useAppDispatch` и `useAppSelector`
- Типизация пропсов, хуков, компонентов, селекторов и ответов API
- Защита от ошибок за счет раннего выявления проблем на этапе компиляции

### 2. Централизованная логика запросов (Redux AsyncThunk)

Вся работа с данными вынесена из компонентов в Redux. Используется createAsyncThunk для параллельного получения списка товаров и общего количества элементов (totalCount) через Promise.all. Это делает компоненты "глухими" к реализации API и упрощает обработку состояний loading / success / error.

### 3. Умная синхронизация с URL

При изменении категории, страницы или типа сортировки, параметры автоматически "вшиваются" в URL. Это позволяет пользователю:

- Вернуться на ту же страницу при обновлении браузера.
- Поделиться ссылкой с уже примененными фильтрами.
- Корректно использовать кнопки "Назад/Вперед" в браузере.

### 4. Серверная фильтрация и поиск

Поиск реализован на стороне сервера (через API параметры), что снижает нагрузку на клиент и позволяет работать с большими объемами данных. Для предотвращения спама запросами внедрен debounce на ввод текста.

### 5. LocalStorage для корзины

Данные корзины сохраняются в localStorage браузера, что позволяет:

- Сохранять добавленные товары после перезагрузки страницы
- Восстанавливать состояние корзины при повторном визите

Реализация включает:

- Утилиту `getCartFromLS` для загрузки корзины из localStorage при инициализации приложения
- Хелпер `calcTotalPrice` для централизованного расчета итоговой стоимости
- Интеграцию с Redux slice для бесшовного управления состоянием

## Возможности

- Каталог: Фильтрация по категориям (Мясные, Вегетарианские и др.).
- Сортировка: По цене, популярности или алфавиту.
- Поиск пицц по названию.
- Пагинация: Постраничный вывод товаров (по 8 позиций на страницу).
- Корзина: Добавление товаров, удаление и автоматический подсчёт итоговой суммы и количества.
- Skeleton UI: Приятная визуализация процесса загрузки карточек.
- Адаптив: Полная поддержка мобильных устройств.

## Запуск

1. Клонируйте репозиторий:

```bash
   git clone [https://github.com/SofiaBazovaya/React-Pizza-Store.git]

```

2. Установите зависимости:

```bash
   npm install
```

3. Запустите сервер для разработки:

```bash
   npm run dev

```

4. Сборка проекта:

```bash
   npm run build
```
